Git 在做什麼？

從單機RPG遊戲的角度想像

  1. 玩遊戲碰到很強的王之前（或者掉寶率很低），通常會存檔 （commit）
  2. 遊戲玩到後期發現前期有一個寶物沒拿，但現在等級太高解不到，希望回到過去解寶物，主線劇情又不想要重打 （checkout、merge）
  3. 遊戲有多種結局，到了叉路不知道該怎麼選 （branch）

Git 是版本控管軟體，像是遊戲存檔的管理，隨時都可以回到過去每一個記錄點，從那邊開始，甚至可以跟你最新的存檔結合。

Git 跟著做

環境建置

首先要在 ubuntu 環境下建置 git 環境

> sudo apt-get install git-core 
> git --version  (安裝後查看 git 版本)
git version 1.7.8.3

設定 config

使用 git 的第一步設定你的名字，這樣在你每次存檔（commit）的時候，別人才知道這個記錄是你玩的

> git config --global user.name 大中天
> git config --global user.email xxx@ooo.com

當設定好後就會在你的 ~/.gitconfig 文件檔加入剛剛的設定 如果你沒有使用 --global 參數的話

> git config user.name 大中天
> git config user.email xxx@ooo.com

那這個設定就會被加在你當前工作目錄的 ./.git/config 中，只有在這個倉庫（Repository）才會套用這個設定。（或者你可想只有在這個專案或遊戲的存檔會用你的名字）

> git config --list

你可以使用上面的指令來看你目前的設定

為了讓介面顯示更漂亮，可以在設定裡面設定呈現使用彩色

> git config --global color.ui true

初體驗

什麼都不要管，照著做一次

> git init （這是 git 初始化）
> git status
> （新增一個叫做 hero.txt 的檔案，隨便打點東西在裡面）
> git status
> git add hero.txt
> git status 
> git commit -m "First Commit"
> git status

三種狀態的介紹

  • 過去的版本控管：記錄變化 (存檔和存檔的變化，LV5->LV6改變了多少 HP MP)
  • git：快照 snapshot

git機制（略）請參考推薦書籍

對於任何一個檔案，在 Git 內都只有三種狀態：已提交（committed），已修改（modified）和已暫存（staged）。已提交表示該檔案已經被安全地保存在本地資料庫中了；已修改表示修改了某個檔案，但還沒有提交保存；已暫存表示把已修改的檔案放在下次提交時要保存的清單中。[節錄於 Pro Git 一書，詳見 ref]

練習狀態不斷切換

目的：新增、修改在 working dir 裡面的檔案，裡用下面指令切換他們的狀態

> git status
> git reset
> git reset <檔名> （只回復單一檔案狀態）
> git commit -m XX（如果 reset 後 commit）
no changes added to commit (use "git add" and/or "git commit -a")

請觀察 git status 的訊息，學著看不同的狀態有什麼樣的訊息出現。（裡面提到的 HEAD，代表你現在所在的位置）

如果覺得每次 commit 都還要打很多指令很麻煩，可以用下面的指令，詳細用法多help查看囉！

> git -i
> git commit -a -m "註解" （代表的是 add + commit，但只有 add 修改的，新增的檔案還是要手動 add 一次）

忽略

有些不想要被追蹤的檔案，可以在該目錄底下新增一個 .gitignore 檔案，並且在這個檔案裡面加入不想被追蹤的檔案名稱，再看 git status 就會發現列在 .gitignore 裡面的檔案都不會出現了。（注意：記得要把 .gitignore 檔名加入到 .gitignore 檔案裡唷！）

觀察過往記錄

目的：觀察所有的提交記錄（想要知道有哪些遊戲存檔）

> git log

但是 log 實在有點冗長，可以這樣做

> git log --pretty=oneline （每筆記錄只顯示一行）
> git log --pretty=oneline --max-count=3  （總共只取最後三筆）
> git log --pretty=oneline --since='4 minutes ago'  （用時間篩選）
> git log --pretty=oneline --until='1 hours ago'  （用時間篩選）
> git log --pretty=oneline --author=<提交作者>  （用作者來篩選）

> git log --all --pretty=format:"%h %cd %s (%an)" --since='7 days ago'    （改變呈現格式，格式請自行搜尋）
> git log --pretty=format:"%h %ad | %s%d [%an]" --graph --date=short  （加入樹狀圖形）

alias 別名

目的：每次要看個 log 就要打好多字好麻煩，有沒有什麼更快的方法？

可以設定 alias 別名，設定後只要輸入短短的別名，就可以執行長長的指令

> git config --global alias.hist "log --pretty=format:\"%h %ad | %s%d [%an]\" --graph --date=short"
> git hist  （看結果）

或者你也可以使用 shell 的 alias：

> alias ghist='git log --pretty=format:"%h %ad | %s%d [%an]" --graph --date=short'
> ghist  （看結果）

在提交的版本中跳躍

目的：提交了很多次，想要跳到某個提交版本（想要玩某個存檔）

> git hist  （還記得我們已經 alias 過了嗎？）
* 000a2c8 2012-05-15 | 等級提升為lv2 (HEAD, master) [大中天]
* e37c7d1 2012-05-15 | 新增技能表 [大中天]
* e8d17a1 2012-05-15 | 建立英雄 [大中天]
> git checkout e8d17a1  （後面參數為 hist 裡面查到的 hash） 
........................
HEAD is now at e8d17a1... 建立英雄
> cat hero.txt  （檔案變回剛初始時的樣子了）

先知道怎麼跳就好了，先跳回來當前的紀錄

> git checkout master

master 是主要的支線（畫在白板上，還記得嗎）

Tag 功能

目的：在使用 checkout 的時候打長長一串 hash 太麻煩，而且也不好打

> git tag lv2
> git hist
* 000a2c8 2012-05-15 | 等級提升為lv2 (HEAD, lv2, master) [大中天]
* e37c7d1 2012-05-15 | 新增技能表 [大中天]
* e8d17a1 2012-05-15 | 建立英雄 [大中天]
> git checkout lv2^  （^代表上一個的意思）
HEAD is now at e37c7d1... 新增技能表
> git tag new_skill
> git hist
* e37c7d1 2012-05-15 | 新增技能表 (HEAD, new_skill) [大中天]
* e8d17a1 2012-05-15 | 建立英雄 [大中天]
> git checkout lv2
> git hist
* 000a2c8 2012-05-15 | 等級提升為lv2 (HEAD, lv2, master) [大中天]
* e37c7d1 2012-05-15 | 新增技能表 (new_skill) [大中天]
* e8d17a1 2012-05-15 | 建立英雄 [大中天]

請注意每筆 log 後面的括弧，我們可以用括弧內的任一值當做 checkout 的參數

檔案變化

目的：commit 如此多，想要知道 commit 之間到底有哪些不一樣

（法一）
> git show
（法二）
> git diff (尚未暫存的檔案更新了哪些部份)
（法三）
> git log -p
> git log -p -2 (只顯示最近兩次的更新)

對單一檔案回復

目的：同時幾個檔案，發現其中有一個檔案改爛了，想要回復到之前的版本

（修改 hero.txt）
> git checkout hero.txt
> cat hero.txt （檔案已經回復）
或者那個檔案想要回復到特定版本
> git checkout HEAD^^ hero.txt
> cat hero.txt （檔案已經回復到剛建立 hero.txt 的樣貌）

修正 commit

目的：commit 之後才發現有些小錯誤，可能只是打錯字，改完錯字後只想修正而不另外再 commit 一個新的

（修改 skill.txt）
> git commit -a -m "習得新技能隔山打牛(有錯字)"
（發現錯字，並修正之）
> git commit --amend -a -m "習得新技能隔山打牛"
> git hist
* 8ede676 2012-05-15 | 習得新技能隔山打牛 (HEAD, master) [大中天]
* 000a2c8 2012-05-15 | 等級提升為lv2 (lv2) [大中天]
* e37c7d1 2012-05-15 | 新增技能表 (new_skill) [大中天]
* e8d17a1 2012-05-15 | 建立英雄 [大中天]

回復某版本

目的：想回復到之前的某個版本

> git hist
* 8ede676 2012-05-15 | 習得新技能隔山打牛 (HEAD, master) [大中天]
* 000a2c8 2012-05-15 | 等級提升為lv2 (lv2) [大中天]
* e37c7d1 2012-05-15 | 新增技能表 (new_skill) [大中天]
* e8d17a1 2012-05-15 | 建立英雄 [大中天]
> git revert lv2
（此時會跳出預設編輯器，讓你編輯 commit 的訊息，可以直接離開就好了）
> git hist

* 8ede676 2012-05-15 | 習得新技能隔山打牛 [大中天]
* 000a2c8 2012-05-15 | 等級提升為lv2 (lv2) [大中天]
* e37c7d1 2012-05-15 | 新增技能表 (new_skill) [大中天]
* e8d17a1 2012-05-15 | 建立英雄 [大中天]

revert 跟之前介紹的 checkout 有什麼差別？還記得在前面我介紹 checkout 時用的詞是跳躍，checkout 可以跳躍到任何一個節點，跳到節點後對檔案做的任何修改都會變成一條新分支。而使用 revert 會把回復的動作當成一次的 commit，是直接在你當前的分支繼續長下去。

如果你是想要還原到某個版本，並且刪除掉那個版本之後的版本，那你可以用 git rest --hard

> git hist
* e9c540f 2012-05-15 | Revert "等級提升為lv2" (HEAD, master) [大中天]
* 8ede676 2012-05-15 | 習得新技能隔山打牛 [大中天]
* 000a2c8 2012-05-15 | 等級提升為lv2 (lv2) [大中天]
* e37c7d1 2012-05-15 | 新增技能表 (new_skill) [大中天]
* e8d17a1 2012-05-15 | 建立英雄 [大中天]
> git reset --hard HEAD^
> git hist --all
* 8ede676 2012-05-15 | 習得新技能隔山打牛 (HEAD, master) [大中天]
* 000a2c8 2012-05-15 | 等級提升為lv2 (lv2) [大中天]
* e37c7d1 2012-05-15 | 新增技能表 (new_skill) [大中天]
* e8d17a1 2012-05-15 | 建立英雄 [大中天]

不過使用上請小心，hard reset 真的會把記錄檔案清光光，我個人比較會用 revert，還可以做些記錄！

分支

目的：開發新功能又怕影響到原本的功能，可以開一個分支開發，如果穩定的話再把它合併（merge）回主要的支線（master）

從目前支線的頂端開一個新的支線 #遊戲遇到選對話，想要先玩玩看其一，如果順利就玩下去

  master

多人維護專案 (git 的魅力)

上面介紹的功能，其實較早以前的版本控制系統都有，但是，為什麼 Git 會越來越熱門？原因在於多人開發時，Git 相對比較容易些，Git 先天的架構，讓他可以是分散式的、可以方便管理 branch，過程記錄很完善，重點是它還很快。這對團隊開發來說，是很強大的優勢，不過這可能需要多使用才能比較與體會了。

要如何多人共同開發 git 專案，其實可以架一個 git server 存放專案的 repository，開發成員透過 SSH 或者 HTTP 傳輸取得 repository (pull)，取得後在本機端繼續開發，跟我們前面所做的動作一樣，修改好後再推(push)回去。基本的概念就是這樣。

雖然架一個 git server 非常簡易，但其實也可以考慮使用一些公開的服務。Github就是一個很棒的 Git 平台，你可以免費在這個平台儲存你的專案並與他人分享。

(github) 抓人家的專案

讓我們實際的看看 github 上人家的專案，並且使用 git 抓下來看看吧！

其實 git 本身也是在 github 上面發展的唷！你可以到 github.com/git/git 看到 git 的原始碼。點選網頁上的『Git Read-Only』按鈕，就可以看到一個連結：git://github.com/git/git.git。你可以使用 git clone 語法抓下來。

> git clone git://github.com/git/git.git
（或要自訂資料夾名稱的話）
> git clone git://github.com/git/git.git 自訂資料夾名稱

(github) 把自己的專案放上去

試試看實際把自己的專案放上去

  1. 註冊 Github 帳號
  2. 照著官方的 Set Up Git 來設定 SSH key （請注意作業系統，官方教學有分成 mac, win, linux 三種）
  3. 照著官方的 Create A Repo 來新建一個 repository。其實在新增一個新的 repo 後，就會在該 repo 頁面看到 step by step 的教學，跟著做就好了。試著把剛剛練習的東西放上去（反正只是文字檔，別害羞XD）

(github) fork and pull request

實際抓一個人家的專案，修改，再上傳

我們常常可以看到有些網站上面可能有個很大的 Fork me on Github 圖片。Fork 這個動作簡單來說，會把這個專案複製一份到你的帳號，你可以直接在你帳號下對這個專案維護修改。當你覺得你改得不錯，想要貢獻回去讓你改的功能變成主版本的功能，你可以對原作者發送一個請求(pull request)，請作者將你的程式碼 pull 回去，如果作者也覺得這功能不錯，原作者可以選擇 merge。

官方有 Fork A Repo 教學，不過其實點一下 fork 按鈕就好了。課堂上大家可以互相 fork 彼此的專案來玩玩看，修改人家的文件，並且發發看 pull request。官方也有一個 Send pull requests 教學可以參考。

你現在看的這份文件也是在 github 上面唷！你可以在這邊 fork 回去，但我應該是不會接受 pull request :)

小結

第一次學習 git 的人可能會對這次的教學感到暈頭轉向，不過還是希望學弟妹能硬撐著，如果你要待在寫程式領域發展，git 是一個你總有一天要碰的東西，而且它真的不難，常用的指令大概才五個而已，只要你開始習慣一個程式寫了一個版本就 git commit -a -m 一下，之後碰到問題，相信你一下就可以從網路上找到解答XD

有很多人寫很漂亮的 git 圖形化工具，我也推薦大家去使用，但是基本的 CLI 指令還是要知道，因為即使使用圖形化介面還是多少還是會用到 CLI 指令。

Git 挑戰

如果想要測試一下自己 Git 學得精不精，有一個用 Ruby 寫的文字介面遊戲叫做 Githug，有點像我們擅長玩的 wargame，大家可以挑戰看看。

因為上週沒有講到 Ruby 這個程式語言，來簡易介紹一下如何使用它。
ruby 有很多版本，在測試時我們常用一個東西叫做 rvm(Ruby Version Manager)，它可以隨時切換不同的 ruby 版本，而且它是將 ruby 裝在自己的家目錄底下，並不會影響系統的 ruby 版本，算是很方便的工具。

請按照 RVM 官方網站的安裝步驟，來裝，才四個步驟而已。如果你電腦裡面沒有 curl 就需要安裝一下 sudo apt-get install curl。

安裝好後敲敲看 which ruby，如果顯示的目錄是在你的家目錄，那你應該就是成功了！
